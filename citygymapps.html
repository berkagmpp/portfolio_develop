CityGym Membership Management Application

In July 2021, I designed a membership management web application for City Gym staff for membership registeration, search, modify, select additional, register, modify, and pay for the fitness class.

• UI / USABILITY (PROTOTYPE)

I referred to Jakob Nielsen: 10 Usability Heuristics for User Interface Design (Nielsen, 2020) and then applied the principles to my initial prototype.

#8: Aesthetic and minimalist design

• As I usually write and watch the web, this prototype is made by completely web design style.

• The application is designed with modern dark mode, which is a trend in the web field these days (Cardello, 2020). Primarily, used minimal colours of the dark Gray family to have minimal & aesthetic design.

#4: Consistency and standards

• The overall layout is classified according to consistency and industry standards. It also splits the panel by subtle colour differences, allowing users to intuitively view the entire configuration, eventually making it easier to recognise.

#2: Match between system and the real world

• I adopted a sequence and shape that was not too different from the real-world paper registration form.

• The format is similar to the form of the paper version, but this form posts the registration cost information at the top right, unlike it is located at the bottom in paper form. The upper right corner is the optimal position for the user to recognise the structure of the form when scanning the screen (Jerry Cao, 2015, p. 20). The registration cost information is shown as a sticky that cannot be implemented in a paper version, allowing follow the screen even if a scroll bar is created.

#6: Recognition rather than recall / #7: Flexibility and efficiency of use

• Breadcrumb placed at the top shows the current user location in the application and makes it easier for the user to navigate and increase freedom by connecting easily to upper categories (Gube, 2009). This location-based Breadcrumb makes it easy to use this application, even if it will have multiple levels of site structure through many categories or functions are added later. And the cost display space on the right side, which is the outcome of the automatically calculated cost, is divided into the price of each item and the final price, increasing user recognition (Gube, 2009).

#1: Visibility of system status

• The progress indicator was also placed at the top to inform steps and current status to complete the registration. I will use the chunking technique: minimizing the overload of information that users need to enter by showing users the big picture in advance by grouping many data into a large structure (Babich, Web page design: A comprehensive guide, 2020; Moran, 2016). This Status Bar shows the present status to allow open and continuous communication between the user and the application.

#5: Error prevention / #10: Help and documentation

• Each large item has a guided note with small letters like required, select, multiple choices or automatic. It intended to make it more accurate and more accessible for users to understand how to use the registration (Babich, How to write and design user-friendly error messages, 2020). It supplies immediate help without requiring user memory.

#9: Help users recognise, diagnose, and recover from errors

• Suppose the user is not aware of the guided notes or the user did not answer the required value or enter the wrong value into each item. In that case, the user pressed the Calculate button; the application provides an error solution with bold red texts and value boxes instinctively raising the user's alertness. It informs the user of the error as the final outcome (Shamonsky, 2014).

#3: User control and freedom

• If the user chooses or enters all incorrectly, the user can control fast and easily via the Reset button next to the Save button without erasing one by one. The Reset button has a relatively less noticeable colour and smaller size than the Save button, preventing unintentional pressing (Babich, Web page design: A comprehensive guide, 2020).

• When the Reset button and the Save button are pressed, the confirmation dialogue window asks the user once more whether to reset/save or cancel the data entered and then executed. It reduces involuntary mistakes, providing efficiency and convenience to users (Yifrah, 2019).
